# SUT 武器資產管理與通報系統

## 一句話描述

透過 NFC/QR Code + Google Apps Script，實現 SUT 武器的資產管理與即時故障通報。

---

## 系統架構

```
NFC 貼紙（每張燒入唯一 SN）
  ↓ 手機感應 / 掃描
瀏覽器開啟 Google Apps Script Web App
  ↓ URL 帶入 ?sn=SUT-W-0001
HTML 前端頁面（手機瀏覽器執行）
  ├─ 辨識模式：顯示該武器完整資訊
  └─ 通報模式：填寫故障/異常表單 → 送出
  ↓
Google Apps Script 後端
  ↓
Google 試算表（資料庫）
  ├─ 資產清單（所有武器主檔）
  ├─ 維修紀錄
  └─ 通報紀錄
```

---

## 核心概念：唯一 SN 值

- 每件武器分配一組 **唯一序號（SN）**，格式待定（例：`SUT-W-0001`）
- SN 燒錄進 NFC 貼紙，貼在武器上
- **SN 是整個系統的核心關聯鍵**：掃描 NFC → 取得 SN → 查詢資產表 → 得到武器所有資訊
- 不管是維修紀錄、通報紀錄、設備清單，全部透過 SN 串接

## 技術棧

| 層級 | 技術 | 說明 |
|------|------|------|
| 前端 | HTML / CSS / JS | 跑在使用者手機瀏覽器 |
| 後端 | Google Apps Script | doGet() 回傳頁面、doPost() 處理資料 |
| 資料庫 | Google 試算表 | 資產清單、維修紀錄、通報紀錄 |
| 觸發介面 | NFC 貼紙 / QR Code | 燒入帶 SN 參數的 URL |
| 圖片資源 | Google Drive | 外連圖片，設公開權限 |

## 功能模組

### 1. 資產管理表
- 所有 SUT 武器的完整清單
- 包含 SN、武器名稱、類型、位置、狀態等欄位
- 作為系統主檔，其他功能都參照此表

### 2. 通報系統
- 掃 NFC → 自動帶入 SN → 顯示武器資訊
- 使用者選擇通報類別（故障、異常、其他）
- 填寫描述 → 送出 → 寫入試算表

### 3. 武器辨識
- 掃 NFC → 顯示該武器的完整資訊
- 不需要填表，純查詢用途
