# SUT 武器資產管理與通報系統 — 需求規格書

<!-- 最後更新: 2026/03/20 -->

---

## 1. 專案目標

為 SUT 所有武器建立一套 **資產管理 + 維修通報** 系統，透過 NFC 貼紙實現：

- **快速辨識**：一掃就知道這是哪件武器、目前狀態
- **快速通報**：武器出問題時，現場人員能立即回報，不需要手動查編號
- **統一管理**：所有武器資料集中在 Google 試算表，方便匯出、統計、追蹤

---

## 2. 使用者角色

| 角色 | 使用情境 |
|------|----------|
| 現場使用者 | 掃 NFC → 查看武器資訊 / 通報維修 |
| 管理者 | 查看試算表 → 追蹤庫存狀態、維修紀錄；收 Gmail 通知 |

## 4. SN 編碼規則

- **格式**：`S` + 5 位數流水號（例：`S00001`）
- **前綴 `S`**：代表 SUT 系列
- **範圍**：`S00001` ~ `S99999`（最多 99,999 件）
- **唯一性**：每件武器一組 SN，不重複
- **用途**：燒入 NFC 貼紙，作為全系統的關聯主鍵

> 武器名稱、類型等資訊不編進 SN，全部由庫存試算表查詢帶入。

---

## 5. NFC / URL 設計

**核心原則：URL 只帶 SN，其他全由後端 + 試算表處理。**

```
https://script.google.com/macros/s/.../exec?sn=S00001
```

- NFC 內容極簡，未來改前端、改武器資訊都不影響 NFC 貼紙
- 新增 / 修改武器 → 只改試算表，不需動 HTML 或 Apps Script
- NFC 貼紙規格：NTAG213（144 bytes）即可

---

## 6. 核心設計：試算表即資料庫

**「庫存狀態」試算表是整個系統的索引中心。**

```
使用者掃 NFC（?sn=S00001）
  ↓
Apps Script doGet() 收到 SN
  ↓
去「庫存狀態」試算表查 SN → 找到對應的武器名稱、類型、狀態
  ↓
回傳 HTML 頁面，顯示該武器資訊 + 通報表單
```

**好處：**
- 新增武器 → 在試算表加一列 + 燒一張 NFC，完事
- 改武器名稱 → 改試算表那一格，立即生效
- HTML / Apps Script 完全不需要動

## 7. 功能需求

### 7.1 庫存狀態試算表（資產主檔 / 索引）

| 欄位 | 說明 | 範例 |
|------|------|------|
| SN | 唯一序號（主鍵） | `S00001` |
| 貼紙編號 | 武器上的實體貼紙號碼 | `1` ~ `4` |
| 藍芽名稱 | 武器的藍芽裝置名稱 | `BT_G22_1`、`BT_M4A1_2`、`BT_HK416_3` |
| 使用槍枝 | 搭配的槍枝型號 | `BOLT` / `KWA HK416` / `G22` |
| 狀態 | 使用狀態 | `展示` / `維修中` / `測試區使用` |
| 存放位置 | 目前放在哪裡 | `大安` / `蘆洲` / `展示` |
| 備註 | 其他補充 | — |

---

### 7.2 頁面功能 A：更新設備狀態

掃 NFC 後可直接更改設備的狀態與存放位置，**即時寫回庫存試算表**。

```
┌─────────────────────────────┐
│  BT_G22_1                   │  ← 藍芽名稱（主標題）
│  SN：S00001                 │
│  貼紙編號：1                 │
│  使用槍枝：G22               │
│  狀態：展示                  │
│  存放位置：大安               │
├─────────────────────────────┤
│  更新設備狀態                 │
│  狀態：○ 展示  ○ 維修中  ○ 測試區使用│
│  位置：○ 大安  ○ 蘆洲  ○ 展示 │
│         [ 更新狀態 ]          │
├─────────────────────────────┤
│            — 或 —            │
```

---

### 7.3 頁面功能 B：維修通報

| 欄位 | 類型 | 說明 |
|------|------|------|
| SN | 自動帶入（唯讀） | 從 URL `?sn=` 取得 |
| 藍芽名稱 | 自動帶入（唯讀） | Apps Script 查庫存表帶入 |
| 緊急程度 | 單選 | `緊急` / `不緊急` |
| 問題項目 | **複選** | `電池充不進去` / `外觀破損` / `感測效果不佳` / `零件鬆脫` / `其他` |
| 補充描述 | 文字輸入 | 選填 |
| 通報人 | 文字輸入 | 必填 |
| 通報時間 | 自動產生 | 系統自動記錄 |

### 7.4 維修通報試算表（紀錄）

每筆通報自動寫入一列：

| 欄位 | 說明 |

| 緊急程度 | 緊急 / 不緊急 |
| 問題項目 | 可複選，以頓號分隔 |

## 8. 系統架構圖

```
NFC 貼紙（?sn=S00001）
  ↓
手機瀏覽器
  ↓
Google Apps Script Web App
  ├─ doGet(sn)
  │   ├─ 查「庫存狀態」試算表 → 取得武器資訊
  │   └─ 回傳 HTML 頁面（顯示武器資訊 + 通報表單）
  │
  └─ doPost()
      ├─ 接收表單資料 → 寫入「維修通報」試算表
      └─ 發送 Gmail 通知管理者
  ↓
Google 試算表
  ├─ 庫存狀態（主檔 / 索引）
  └─ 維修通報（紀錄）
```
